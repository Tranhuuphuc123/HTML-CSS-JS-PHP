Luồng hoạt đông của laravel project

1/ file .env: thiết lập cấu hình data sql
2/ index.php: đây là trang chính  chạy ứng dụng nới bắt đầu ứng dụng của mình
  + xác định gọi controler nào
 -> controller tiếp nhận yêu cầu -> gọi đến các model , view để xử lý
-> phản hồi ngc lại cho index resp cho client

 + require/incluce: dùng để chèn hay load thư viện, gọi các lib bổ trợ
-> dùng nó để thực thi chức năng trên
 +  trang index tiếp nhận dữ liệu và chứa trong biến $request, tất cả dl từ client gửi lên đều chứa trong index
 
3/  các phương thức trong file web.php:
   ++ delete: xóa dữ liệu
   ++ get: show dữ liệu
   ++ post: Save dữ liệu
   ++ put: update
   ++ destroy: để hủy giao diện
   ++ middleware: 
   ++ /admin/dashboard: dùng để chỉ trang dashbỏad chỉ cho user admin sử dụng thui

==>file web.php dùng để cấu hình quy định các controller nào đc xử lý để gửi resp, req cho client, giúp truy cập đc giao diện chức năng mong muốn.

4/ CustomerController:
 + xử lý các 
   ++ index: là method hiển thị lên
   ++ create: khởi tạo
   ++ store: lưu dữ liệu lại, lấy dữ liệu thông qua request(chứa thông tin do client submit lên)

5/ Model: file Customer
 - $fillable: ánh xạ các member variable/properties bên sql qua project laravel


6/ Resource/View: xây dựng giao diện hiển thị
  - trong code php giao diện: 
    + name: lưu ý nó lấy tên đc  ánh xạ đúng tên bên controller đúng với tên bên models đc ánh xạ đúng với tên từ sql

7/folder migrations: nơi dùng để tạo các file dùng để tạo, load các bảng cho cơ sở dữ liệu ánh xạ qua project laravel
 => quản lý các thay đổi trongg cơ sở dữ liệu, lưu trữ các thay đổi
